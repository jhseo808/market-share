### 권장사항

1. 【최우선 테스트 조합 정의】 Chrome(Android 16.0·15.0) + Samsung Internet 29.0(Samsung 기기) + Safari iPhone(iOS 18.6·18.5) 3개 조합을 핵심 스모크 테스트 스위트로 구성하여 전체 사용자의 약 80% 이상 커버리지 확보.
2. 【Whale Browser 전용 테스트 체크리스트 운영】 네이버 Whale의 사이드바 패널 오버레이, 스플릿뷰 레이아웃 깨짐, 팝업·광고 차단 정책에 의한 기능 오작동, 네이버 계정 SSO 연동 시나리오를 별도 수동 테스트 케이스로 관리하고 매 릴리스마다 실행.
3. 【Android 버전 정책 즉시 갱신】 Android 16.0(56.93%)을 must_test 1순위로 즉시 편입하고, 기존 Android 14.0 중심 테스트 환경을 Android 16.0·15.0·13.0 3-tier 구조로 재편. 구형 기기(Android 12.0 이하)는 크리티컬 버그 한정 검증으로 범위 축소.
4. 【iOS 버전 데이터 재검증 및 대응 전략 수립】 iOS 26.2 점유율 급등(51.42%)의 데이터 신뢰성을 StatCounter·Firebase Analytics 등 내부 데이터와 교차 검증. 정식 버전 확인 전까지 iOS 18.6·18.5·18.7을 주력 테스트 버전으로 유지하고, iOS 26.x 정식 출시 확인 즉시 테스트 환경 구축.
5. 【Samsung Internet 회귀 테스트 자동화】 Samsung Internet 점유율 하락에도 불구하고 삼성 기기 기본 브라우저 지위를 고려, Appium + Samsung 실제 기기(Galaxy S·A 시리즈)를 활용한 자동화 회귀 테스트를 CI/CD 파이프라인에 통합. 특히 미디어 재생, 결제 플로우, 폼 자동완성 시나리오 집중 검증.
6. 【Edge 브라우저 Windows 환경 테스트 강화】 Edge(8.46%)는 Windows(81.89%) 기본 브라우저로 실질 사용자 비중이 높음. Chromium 기반이나 IE 모드, 엔터프라이즈 정책, 스마트스크린 필터에 의한 기능 차단 시나리오를 should_test 항목으로 포함하고 Windows 11 + Edge 최신 버전 조합 필수 검증.
7. 【실제 기기 테스트 인프라 구성】 에뮬레이터 의존도를 줄이고 Samsung Galaxy(S·A 시리즈), Apple iPhone(최신 2세대), 네이버 Whale 설치 PC 환경을 포함한 실제 기기 테스트 랩 또는 BrowserStack Real Device Cloud를 활용하여 터치 이벤트·하드웨어 가속·폰트 렌더링 등 에뮬레이터 미재현 이슈 검출.
8. 【월간 점유율 모니터링 및 테스트 우선순위 자동 갱신】 StatCounter KR 데이터를 월 1회 수집하여 임계값 기준(must_test >10%, should_test 3~10%)에 따른 테스트 매트릭스를 자동 갱신하는 스크립트를 구축. Android·iOS 버전 세대 교체 속도가 빠른 만큼 분기별 전면 재검토 프로세스를 QA 정례 일정에 포함.
9. 【macOS + Safari 조합 커버리지 확대】 macOS(3.67%) + OS X(2.38%) 합산 약 6%와 Apple 기기 성장 트렌드를 고려하여 Safari on macOS를 should_test 필수 항목으로 격상. WebKit 전용 CSS 속성(-webkit- prefix), input[type=date] UI, 스크롤 관성, Web Crypto API 동작을 체크리스트에 추가.
10. 【저점유율 브라우저 리스크 기반 선별 검증】 Firefox(0.75%)·Opera(0.37%)·Brave(0.35%)는 정기 회귀 대상에서 제외하되, 결제·인증·보안 관련 핵심 플로우에 한해 분기 1회 스팟 테스트 실시. IE(0.09%)는 공식 지원 종료 선언 후 테스트 대상에서 완전 제외하고 IE 접속 시 지원 종료 안내 페이지 노출 여부만 확인.
